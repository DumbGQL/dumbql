## Why Not Apollo, URQL, or Relay?

| Problem | Apollo Client | URQL | Relay | **DumbQL** |
|---|---|---|---|---|
| **Bundle Size** | ~50KB+ gzipped вҖ” heavy default with `graphql-tag` re-exports preventing tree-shaking | ~10KB base, but normalized cache is separate (+8KB) | ~45KB (66KB with React bindings) | **~10KB core**. Modular packages вҖ” only load what you use. No tree-shaking issues |
| **Cache Update After Mutation** | Manual `cache.modify`/`update`/`refetchQueries` on every mutation. Easy to forget, causes stale UI | Auto-invalidates queries with same types вҖ” but can over-invalidate (v7 bug) causing unnecessary network requests | Automatic via updaters + compiler вҖ” but requires strict Connection/Node schema conventions | **Dual strategy**: cache middleware auto-extracts entities and merges them. Mutations evict related cache keys forcing refetch. No manual `update` needed |
| **Normalized Cache Setup** | Complex `typePolicies` with `keyFields`, `merge` functions for every type | Document cache default is too simple. Graphcache requires separate package + schema config | Built-in but requires global ID (Node interface) + Connection spec | **Zero-config**: `__typename` + `id`/`_id` auto-detection. Automatic entity normalization and merge. Optional `typePolicies` for advanced cases |
| **Pagination** | Manual `relayStylePagination()` + custom merge logic per field | No built-in pagination вҖ” custom logic required | Connection spec only вҖ” rigid, non-negotiable | **Built-in cursor + offset pagination** helpers with merge functions. Configurable limit, debounce |
| **Offline Support** | Requires `apollo-cache-persist` (third-party). No mutation queue | Requires `@urql/exchange-graphcache` + storage layer. Mutation queue needs custom logic | `react-relay-offline` adds 16KB. Best offline mutations | **Built-in offline queue middleware**. Queue persistence to localStorage. Auto-replay on reconnect. Configurable max queue size |
| **Auth Token Refresh** | No built-in refresh flow. Requires custom `Link` | `@urql/exchange-auth` | Custom network layer | **Built-in `authRefreshMiddleware`**. Queues pending requests during refresh. Customizable trigger statuses, retry attempts, header name |
| **DevTools** | Apollo DevTools (Google Chrome extension). Requires GraphOS for advanced features | URQL DevTools (basic) | Relay DevTools (basic) | **Built-in browser extension** (Chrome + Firefox). Schema visualization, request timeline, entity inspection, field tree parsing |
| **File Uploads** | Requires `apollo-upload-client` (third-party) | Built-in via `@urql/exchange-multipart` | Not supported | **Built-in `UploadService`**. Full GraphQL multipart request spec. Auto-detection of `File`/`Blob` in variables |
| **Type Safety** | Optional codegen. Runtime type inference | Optional codegen. Simple generics | Compiler-generated (best-in-class but requires build step) | **Typed `DocumentNode`**. Phantom types for result + variables. Custom codegen CLI. Generated types from introspection JSON |
| **SSR** | Full support but manual transfer state setup | Good support | Built-in but React-only | **SSR streaming + `TransferState` integration**. Server-to-browser cache transfer. Platform-aware save/restore |
| **Subscriptions** | Supported via WebSocket (`subscriptions-transport-ws`/`graphql-ws`) | Separate `@urql/exchange-subscriptions` | Limited | **Built-in `GraphqlSubscriptionService`**. `graphql-transport-ws` protocol. Standalone `subscribe()` function. DevTools integration |
| **Error Handling** | `errorPolicy` (`none`/`all`/`ignore`) вҖ” but type narrowing is tricky (`data` can be `undefined`) | Error policy via exchanges. `data: null` on schema mismatch | Error boundaries + Suspense | **Three error policies**. Optional notification service integration (Taiga UI, Material, etc.). Type-safe `GraphQLResult<T>` discriminated union |
| **Reactive Local State** | Reactive variables (since v3) | No built-in. Must use external state | `RelayStore` but server-data only | **`ReactiveVar` / `makeVar()`** вҖ” wraps `BehaviorSubject`. Client directive middleware for `@client` fields. Local resolver support |
| **Persisted Queries (APQ)** | `apollo-link-persisted-queries` (third-party) | `@urql/exchange-persisted` | Built-in (auto-generated by compiler) | **Built-in `apqMiddleware`**. SHA-256 hashing with fallback. Automatic hash registration on `PersistedQueryNotFound` |
| **Retry Logic** | Built-in `RetryLink` (exponential backoff) | `@urql/exchange-retry` | Built-in | **Built-in `retryExchange`** with exponential backoff + jitter. Custom `shouldRetry` callback. Configurable exponent, max delay |
| **Focus Refetch** | Not built-in. Requires custom implementation | `@urql/exchange-refocus` | Not built-in | **Built-in `focusRefetchMiddleware`**. `visibilitychange` + `window.focus` support. Configurable stale time |
| **Request Batching** | `apollo-link-batch-http` | Not built-in | Third-party | **Built-in batching**. Configurable `batchWindow` (default 50ms). Automatic batch flush |
| **Request Deduplication** | Built-in | Not built-in | Not needed (compiler) | **Built-in dedup**. `shareReplay(1)`-based. Automatic cache key management |
| **Testing** | `MockedProvider` + `MockLink` вҖ” complex setup | `mockExchange` | `RelayMockEnvironment` вҖ” steep learning curve | **`MockGraphqlService`**. Simple `when(query, result)` API. FIFO response queue. Optional simulated delay |
| **Debugging** | Apollo DevTools only | DevTools (basic) | DevTools (basic) | **`GraphqlDebugService`** + full debug panel component. Request log, field tree parser, timing chart, entity inspector. Built-in component included |
| **Angular Integration** | React-first port (Apollo Angular is a wrapper). Lags behind React version | React-only | React-only | **Angular-native**. Standalone components, Signals-compatible, SSR with `TransferState`, `@Injectable` services, pipes, `ng add` schematics |
| **CLI Setup** | Create client manually | Create client manually | Requires compiler setup | **`ng add @dumbql/core`**. Interactive prompts. Auto-generates `dumbql.config.ts` |
| **Learning Curve** | Moderate (basic) вҶ’ Steep (advanced cache) | Low вҖ” most approachable | Very steep (3-5x Apollo) | **Low**. Familiar `HttpClient`-based. Intuitive discriminated union results |

## Features Overview

### `@dumbql/core` вҖ” Core (~10KB)
- `GraphqlService` вҖ” query, mutate, refetch, poll, setEndpoint
- Middleware pipeline вҖ” auth, logging, devtools, cache
- Standalone functions вҖ” `query()`, `mutate()`, `refetch()`, `poll()` (inject-free)
- Result helpers вҖ” `isSuccess`, `isError`, `unwrap`, `unwrapOrThrow`, `mapResult`
- `gql` tag вҖ” `parse()`-based DocumentNode creation
- `TypedDocumentNode<TResult, TVars>` вҖ” phantom-typed documents
- Config system вҖ” `DumbqlConfig` with typed sub-configs
- Angular pipes вҖ” `GqlPipe`, `GraphqlDataPipe`, `GraphqlErrorPipe`
- Reactive variables вҖ” `makeVar<T>()`, `ReactiveVar<T>`
- Client directive middleware вҖ” `@client` field resolution
- Router integration вҖ” `guardedRoute()`, `provideDumbqlRouter()`
- Auto-refetch вҖ” `mutationCachePolicy()`, `provideAutoRefetch()`
- DevTools integration вҖ” `DevtoolsService`, `devtoolsMiddleware`, `provideDevtools()`
- Schema service вҖ” `SchemaService`, `provideSchemaFetch()`
- DumbQL plugin system вҖ” `DumbqlPlugin` with `onInit` + `getMiddleware`
- Request batching, deduplication, retry with configurable policy
- Error notification integration (callback or service-based via DI)
- `ng add` schematics вҖ” interactive setup

### `@dumbql/cache` вҖ” Normalized Cache
- `NormalizedCache` вҖ” in-memory entity store keyed by `__typename:id`
- `CacheService` вҖ” Angular service with CRUD + local reactive state
- Entity auto-extraction вҖ” `extractEntities()` recursively walks response data
- Cache middleware вҖ” query caching, mutation eviction, entity merging
- Garbage collector вҖ” reference-counting with configurable TTL
- Persistence вҖ” `localStorage` with throttling, versioning, max age
- Optimistic updates вҖ” snapshot/rollback/commit

### `@dumbql/subscriptions` вҖ” WebSocket Subscriptions
- `GraphqlSubscriptionService` вҖ” `graphql-transport-ws` protocol
- Standalone `subscribe()` function
- Auto WebSocket URL derivation from HTTP endpoint
- Connection init/ack handshake, 10s timeout
- DevTools messaging integration
- Lazy connect, reconnect support

### `@dumbql/file-upload` вҖ” Multipart File Uploads
- `UploadService` вҖ” GraphQL multipart request spec
- Auto-detection of `File`/`Blob` in nested variables
- FormData construction with operations + map
- Configurable max file count and size

### `@dumbql/middlewares` вҖ” Middleware Plugins
- `authRefreshMiddleware` вҖ” 401 intercept + token refresh + request queue
- `retryExchange` вҖ” exponential backoff with jitter
- `focusRefetchMiddleware` вҖ” refetch on tab focus
- `offlineQueueMiddleware` + `OfflineQueueService` вҖ” offline mutation buffering + auto-replay

### `@dumbql/pagination` вҖ” Pagination Helpers
- `offsetPagination()` вҖ” stateful offset-based pagination with loadMore/refresh
- `cursorPagination()` вҖ” relay-style cursor pagination
- Merge functions вҖ” `offsetMerge()`, `cursorMerge()` for type policies

### `@dumbql/persisted-queries` вҖ” Automatic Persisted Queries
- `apqMiddleware` вҖ” SHA-256 hash, automatic registration
- Simple hash fallback when `crypto.subtle` unavailable

### `@dumbql/fragments` вҖ” Fragment Utilities
- `fragment()` template tag вҖ” typed FragmentDefinition
- `spread()` / `compose()` вҖ” fragment composition
- `useFragment()` вҖ” type-safe data accessor

### `@dumbql/ssr` вҖ” Server-Side Rendering
- `SsrStreamService` вҖ” chunked TransferState with configurable prefix
- `TransferCacheService` вҖ” server-to-browser cache transfer
- Platform-aware save/restore

### `@dumbql/debugging` вҖ” Debug & Inspection
- `GraphqlDebugService` вҖ” automatic operation recording (500 entry ring buffer)
- `parseFieldTree()` вҖ” query string вҶ’ field tree
- `buildMutationChart()` вҖ” timing visualization data
- `normalizeData()` / `groupEntities()` вҖ” entity inspection

### `@dumbql/downloader` вҖ” Schema Download
- `downloadAndStoreSchema()` вҖ” introspection вҶ’ JSON + SDL files
- CLI integration via `tools/download-schema.mjs`
- Custom headers support

### `@dumbql/testing` вҖ” Testing Mock Backend
- `MockGraphqlService` вҖ” register responses with `when(request, result)`
- FIFO response queue, optional simulated delay
- `provideDumbqlTesting()` вҖ” test provider setup

## Quick Start

```bash
# Create a new Angular project
ng new my-app --standalone

# Add DumbQL (interactive prompts)
ng add @dumbql/core

# Or manual install
npm install @dumbql/core @dumbql/cache
npm install -D @dumbql/downloader

# Start dev server
npm start
```

### Minimal Setup

```typescript
// app.config.ts
import { provideDumbql } from '@dumbql/core';
import { provideHttpClient } from '@angular/common/http';

export const appConfig = {
  providers: [
    provideHttpClient(),
    provideDumbql({
      endpoint: 'http://localhost:4000/graphql',
      // Optional: cache, subscriptions, middlewares, etc.
    }),
  ],
};
```

### Usage

```typescript
import { Component, inject } from '@angular/core';
import { GraphqlService, gql } from '@dumbql/core';
import { AsyncPipe } from '@angular/common';
import { Observable, map } from 'rxjs';

const GET_USER = gql`{ getUser { id name email } }`;

@Component({
  selector: 'app-user',
  standalone: true,
  imports: [AsyncPipe],
  template: `<div>{{ (user$ | async)?.name }}</div>`,
})
export class UserComponent {
  private gql = inject(GraphqlService);
  user$ = this.gql.query(GET_USER).pipe(
    map(r => r.status === 'success' ? r.data.getUser : null),
  );
}
```

## Configuration Reference

```typescript
// dumbql.config.ts
import type { DumbqlConfig } from '@dumbql/core';

const config: DumbqlConfig = {
  // в”Җв”Җ Core в”Җв”Җ
  endpoint: 'http://localhost:4000/graphql',
  errorPolicy: 'none',        // 'none' | 'all' | 'ignore'
  retryCount: 3,
  retryDelay: 1000,           // ms (exponential: 1s, 2s, 4s...)
  dedup: true,                // Deduplicate in-flight queries
  batchWindow: 50,            // Batch window in ms (0 = disabled)
  headers: { Authorization: 'Bearer ...' },
  middleware: [],              // Custom middleware array
  plugins: [],                 // DumbqlPlugin array
  devAuth: { enabled: false },

  // в”Җв”Җ Subscriptions в”Җв”Җ
  subscriptions: {
    wsEndpoint: 'ws://localhost:4000/graphql',
    reconnect: true,
    reconnectInterval: 2000,
    lazy: true,
  },

  // в”Җв”Җ Cache в”Җв”Җ
  cache: {
    enabled: true,
    maxAge: 300_000,
    serialize: true,
    typePolicies: {
      User: { keyFields: ['id'] },
      PaginatedResult: { merge: 'append' },
    },
  },

  // в”Җв”Җ Persisted Queries в”Җв”Җ
  persistedQueries: {
    enabled: false,
    hash: 'simple',            // 'sha256' | 'simple'
    autoPersist: false,
  },

  // в”Җв”Җ File Upload в”Җв”Җ
  upload: {
    maxFiles: 10,
    maxFileSize: 10_485_760,   // 10MB
  },

  // в”Җв”Җ Debug в”Җв”Җ
  debug: {
    logOperations: true,
    logTiming: true,
    logCache: false,
```

## Error Handling Pattern

```typescript
import { isSuccess, isError, unwrap, unwrapOrThrow } from '@dumbql/core';

service.query<{ user: User }>(GET_USER).subscribe(result => {
  if (isSuccess(result)) {
    // result.data is typed
    console.log(unwrapOrThrow(result).user);
  } else if (isError(result)) {
    // result.error is string
    console.error(result.error);
  }
});
```

Or with pipes:
```html
<!-- graphql-data pipe extracts data, null on error -->
<div>{{ result | graphqlData | json }}</div>

<!-- graphql-error pipe extracts error string, null on success -->
<div *ngIf="result | graphqlError as err">{{ err }}</div>
```

## Bugs Fixed in Other Clients

Real GitHub issues from Apollo, URQL, and Relay that DumbQL addresses by design.

| Project | Issue | Problem | DumbQL Fix |
|---|---|---|---|
| Apollo | [#9319](https://github.com/apollographql/apollo-client/issues/9319) | `INVALIDATE` in `cache.modify` does not evict data вҖ” stale data persists with no refetch | Cache middleware auto-evicts entities on mutation. No manual `modify`/`evict` needed |
| Apollo | [#10289](https://github.com/apollographql/apollo-client/issues/10289) | `cache.evict` silently no-ops inside optimistic UI вҖ” open since 2022 | Optimistic updates are snapshots with explicit commit/rollback вҖ” eviction always works |
| Apollo | [#11804](https://github.com/apollographql/apollo-client/issues/11804) | Skipped query returns outdated data after `clearStore()` вҖ” cache reset is ignored by skipped hooks | No skip concept. Queries are imperative вҖ” reset always returns fresh state |
| Apollo | [#9735](https://github.com/apollographql/apollo-client/issues/9735) | Production-only bug: internal results cache merges stale data into `readFromStore` output | No internal result cache вҖ” every read goes directly to the normalized store |
| Apollo | [#8958](https://github.com/apollographql/apollo-client/issues/8958) | `@apollo/client` requires `react` as dependency even in non-React projects | Framework-agnostic core (`@dumbql/client`) has zero framework dependencies |
| Apollo Angular | [#2371](https://github.com/the-guild-org/apollo-angular/issues/2371) | apollo-angular incompatible with `@apollo/client` v4.0 вҖ” Angular version lags behind React | Angular packages track core in lockstep вҖ” no React version to wait for |
| URQL | [#2414](https://github.com/urql-graphql/urql/issues/2414) | `relayPagination` does not reset data when non-relay params change вҖ” shows stale results | Pagination helpers are stateless вҖ” variable changes always produce a clean slate |
| URQL | [#668](https://github.com/urql-graphql/urql/issues/668) | Query with `relayPagination` does not refetch when variables change вҖ” returns stale data | Query refetch on variable change is guaranteed вҖ” no stale data regression |
| URQL | [#3877](https://github.com/urql-graphql/urql/pull/3877) | `relayPagination` concatenates pages in cache-write order, causing flickering mis-ordered items | Cursor merge functions use explicit ordering вҖ” no dependency on write timing |
| Relay | [#3406](https://github.com/facebook/relay/issues/3406) | Relay is React-only. No Angular, Vue, or Svelte support вҖ” framework lock-in | DumbQL ships first-class bindings for Angular, React, and Vue from day one |
| Relay | [#183](https://github.com/facebook/relay/issues/183) | Relay mandates `Node` interface + `Connection` spec вҖ” backend must conform | Supports offset, cursor, and relay-style pagination вҖ” no backend changes required |
