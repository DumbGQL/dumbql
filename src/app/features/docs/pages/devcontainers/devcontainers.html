Sandboxes

Try DumbQL instantly — online via StackBlitz or locally with Dev Containers. Each sandbox includes a running mock GraphQL backend and a minimal app that queries it using @dumbql/* packages.

StackBlitz (Online)

Click a link below to open a fully-configured sandbox in your browser. No installation required.

Sandbox Stack Link
Angular Angular 22 + all @dumbql/* packages stackblitz.com/~/github.com/DumbGQL/dumbql/tree/main/demos/angular
React Vite + React 19 + @dumbql/react stackblitz.com/~/github.com/DumbGQL/dumbql/tree/main/demos/react
Vue Vite + Vue 3 + @dumbql/vue stackblitz.com/~/github.com/DumbGQL/dumbql/tree/main/demos/vue

Every sandbox includes a mock GraphQL endpoint at /graphql. The @dumbql/* packages are installed from npm.

VS Code Dev Containers

For local development with Docker, each sandbox is available as a Dev Container configuration:

Sandbox Config Ports
Angular .devcontainer/angular/devcontainer.json 4200 + 4000
React .devcontainer/react/devcontainer.json 5173 + 4000
Vue .devcontainer/vue/devcontainer.json 5173 + 4000 
# 1. Open the config directory in VS Code
code .devcontainer/react/

# 2. Press Ctrl+Shift+P → "Dev Containers: Reopen in Container"
# VS Code builds the container, installs deps, and starts both servers.
